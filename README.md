# Autonomous DeFi Agent

Autonomous DeFi agent that snipes jobs in real-time, generates AI-powered Solidity code, auto-responds to clients, and orchestrates other agents on [market.near.ai](https://market.near.ai).

Built for the **[COMPETITION] Build the Most Useful Agent** — 100 NEAR prize pool.

---

## What Makes This Agent Different

Most agents on the marketplace are generic bid-spammers. This agent is a **DeFi specialist** with five integrated capabilities:

### 1. Sniper Mode (Real-Time SSE)
While other agents poll every 30 minutes, this agent listens to the marketplace **SSE feed in real-time** and bids within seconds of a new job appearing.

```
[SSE] New job: 'Audit my Solidity contract' (5 NEAR)
  -> SNIPED! Bid 4.2 NEAR (score=0.72) in 3.2s
```

### 2. AI Brain (Claude-Powered)
Every proposal and deliverable is generated by **Claude AI**, tailored to each specific job. No two proposals are the same.

- **AI Proposals** — Reads the full job description and writes a personalized bid referencing specific technical approaches
- **AI Code Generation** — Generates complete, compilable Solidity contracts customized to job requirements
- **Template Fallback** — If AI is unavailable, falls back to battle-tested Solidity templates

```
Template proposal: "Web3 developer specializing in Solidity..." (generic, 166 chars)
AI proposal:       "I've deployed Uniswap V4 hooks on Sepolia with TWAP-based
                    dynamic fees—I'll apply the same pattern here..." (tailored, 774 chars)
```

### 3. Team Lead Mode (Multi-Agent Orchestration)
For complex jobs (budget > 8 NEAR), the agent acts as a **team lead**:

1. Breaks the job into sub-tasks (frontend, tests, docs)
2. Posts sub-tasks as new jobs on the marketplace
3. Does the core Solidity work itself
4. Combines all deliverables into a final submission

This creates a **network effect** — the agent uses the marketplace to deliver better results on the marketplace.

### 4. Messenger (Auto-Revision)
The agent **monitors client messages** and responds automatically:

- **Revision requests** — AI generates updated deliverables incorporating feedback
- **Questions** — AI answers technical questions about the code
- **Approvals** — Responds professionally and closes the loop

No more ghosting clients. The agent is always online.

### 5. Memory (Learning System)
Persistent tracking across restarts:

- Records every bid, win, loss, and delivery
- Tracks earnings and win rate over time
- Analyzes which tags/skills win the most jobs
- Avoids re-bidding on jobs already bid on (even after restart)

```
Memory: 36 bids tracked, 2 wins, 56% win rate, 25.6 NEAR earned
Top tags: solidity (2 wins), uniswap (2 wins), defi (2 wins)
```

## Architecture

```
agent.py                 Main orchestrator — runs cycles, coordinates all modules
├── brain.py             AI engine — Claude API for proposals, code gen, revisions
├── sniper.py            SSE listener — real-time job detection (background thread)
├── skills.py            Job scoring, bid calculation, AI proposal generation
├── code_generator.py    AI + template Solidity code generation
├── messenger.py         Auto-reply to client messages and revision requests
├── memory.py            Persistent bid/win/earnings tracking (JSON)
├── team_lead.py         Job decomposition and multi-agent delegation
├── market_client.py     Full API client for market.near.ai (REST + SSE)
└── config.py            Skills, budgets, AI keys, and agent configuration
```

## How It Works

```
┌─────────────────────────────────────────────────────┐
│                    SNIPER MODE                       │
│         SSE Feed ──> Instant bid (<5 seconds)        │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│                   POLL CYCLE (every 30 min)           │
│                                                       │
│  1. GET /jobs?status=open                             │
│  2. Score each job (tags + keywords + budget)         │
│  3. Check memory — skip jobs already bid on           │
│  4. AI generates tailored proposals per job           │
│  5. Bid on top 5 matches                              │
│  6. Check accepted bids                               │
│     ├── Simple job ──> AI Code Gen ──> Submit         │
│     └── Complex job ──> Team Lead ──> Delegate + Submit│
│  7. Check client messages ──> AI auto-reply           │
│  8. Check delegated sub-tasks                         │
│  9. Update memory (bids, wins, earnings)              │
└───────────────────────────────────────────────────────┘
```

## Quick Start

```bash
# Clone
git clone https://github.com/tomazzi14/autonomous-defi-agent.git
cd autonomous-defi-agent

# Setup
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Configure
cp .env.example .env
# Edit .env with your Agent Market API key + Anthropic API key

# Run
python agent.py --status    # Check agent status
python agent.py             # Run one cycle
python agent.py --sniper    # Real-time mode (SSE + polling)
python agent.py --demo      # Verbose demo cycle

# Test
python test_brain.py        # Test all modules
python test_brain.py --brain    # Test AI brain only
python test_brain.py --memory   # Test memory only
```

## Scoring Algorithm

Jobs are scored 0.0 to 1.0 based on:

| Factor | Weight | How |
|--------|--------|-----|
| Tag match | 50% | Our skills vs job tags |
| Keyword match | 30% | DeFi/Solidity keywords in title + description |
| Budget | 20% | Higher budget = higher score |

Hard filters (instant skip): spam tags, non-NEAR tokens, descriptions < 20 chars.
